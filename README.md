# GSP Children Portal

WordPress-плагин для лендинга «Детские программы Газстройпрома». Страница выводится шорткодом:

```text
[gsp_children_portal]
```

## Архитектура контента

Плагин больше не использует стандартные записи `post` и стандартные категории `category` для фронтенда.

Контент хранится в отдельном Custom Post Type:

- `gsp_child_item`
- название в админке: **Детские программы ГСП**
- отдельный пункт левого меню WordPress
- поддержка: title, editor, excerpt, thumbnail, page-attributes

Разделы страницы задаются отдельной таксономией:

- `gsp_child_section`
- название: **Разделы страницы**
- используется только для `gsp_child_item`

Термы создаются при активации плагина:

| Slug | Назначение |
| --- | --- |
| `hero` | Главный экран |
| `programs` | Программы и направления |
| `events` | Ближайшие мероприятия |
| `stories` | Истории сотрудников |
| `partners` | Партнёрские предложения |
| `faq` | Вопросы и ответы |
| `materials` | Полезные материалы |
| `footer` | Нижняя плашка |

Все фронтенд-запросы идут только к `post_type => gsp_child_item` и `taxonomy => gsp_child_section`.

## Как добавлять контент

1. Откройте **Детские программы ГСП → Добавить элемент**.
2. Заполните заголовок, описание, краткое описание и иллюстрацию.
3. В блоке **Разделы страницы** выберите нужный раздел: `hero`, `programs`, `events`, `partners` и т.д.
4. Заполните дополнительные поля в meta box:
   - возраст;
   - дата события;
   - дедлайн регистрации;
   - текст кнопки;
   - внешняя ссылка;
   - бейдж;
   - порядок сортировки;
   - формат;
   - имя сотрудника;
   - должность/подразделение;
   - ссылки и тексты hero-кнопок.

## Демо-контент

В админке доступен раздел **Детские программы ГСП → Демо-контент**.

- **Создать демо-контент** — создаёт элементы CPT во всех разделах. Дубли не создаются, проверка идёт по `_gspcp_demo_key`.
- **Удалить демо-контент** — удаляет только элементы с meta `gsp_is_demo = 1`.

Если опубликованных элементов в разделе нет, фронтенд использует runtime fallback-массивы. Они не создают записи в блоге и нужны только для того, чтобы дизайн не разваливался на пустом сайте.

## Миграция старых записей

Автоматической миграции нет.

Для ручной миграции откройте **Детские программы ГСП → Настройки** и нажмите **Мигрировать старые записи**. Кнопка копирует старые записи из категорий `gsp-children-*` в новый CPT и переносит поддерживаемые meta-поля.

## Файлы

- `gsp-children-portal.php` — bootstrap, активация, shortcode, assets.
- `includes/post-types.php` — регистрация CPT.
- `includes/taxonomies.php` — регистрация таксономии и обязательных термов.
- `includes/meta-boxes.php` — meta-поля CPT.
- `includes/queries.php` — выборки для фронтенда.
- `includes/helpers.php` — helper-функции и runtime fallback-контент.
- `includes/admin.php` — страницы настроек, демо-раздел, колонки и фильтры списка.
- `includes/demo-content.php` — создание/удаление демо и ручная миграция.
- `templates/portal-page.php` — HTML лендинга.
- `assets/css/gsp-children-portal.css` — заскоупленные стили `.gspcp-root`.
- `assets/js/gsp-children-portal.js` — плавная навигация.
- `assets/img/*.svg` — fallback-иллюстрации.

## Проверки перед релизом

```bash
php -l gsp-children-portal.php
find includes -name '*.php' -print0 | xargs -0 -n1 php -l
node --check assets/js/gsp-children-portal.js
git diff --check
```

На сайте проверить:

- плагин активируется;
- в меню появился раздел **Детские программы ГСП**;
- можно создать элемент страницы и выбрать раздел;
- meta-поля сохраняются;
- шорткод `[gsp_children_portal]` выводит страницу;
- контент не попадает в общие записи сайта;
- `/dp/` корректно выглядит на desktop и mobile;
- обычный пользователь не видит технические сообщения.
